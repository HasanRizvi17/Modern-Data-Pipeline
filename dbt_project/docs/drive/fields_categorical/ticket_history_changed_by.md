### Ticket History Changed By
Identifies the actor responsible for a change in ticket history.

### Possible Values
| Actor | Meaning |
|------|--------|
| `user` | Change initiated by the user |
| `agent` | Change initiated by a support agent |
| `system` | Change automatically generated by the system |

### Business Rules
- Actor is assigned automatically based on event source
- Value cannot be null

### Usage Notes
- Use this field to audit support workflows
- Analyze automation impact using `system` changes
